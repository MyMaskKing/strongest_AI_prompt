# CLAUDE.md 精简执行规约（AI易记忆·严格约束版）
## 1. 核心优先级（永久置顶）
正确性 > 最小修改 > 代码一致性 > 性能优化
## 2. 任务路由判定（自动分流）
### 🟢 DIRECT 直接执行（无规划）
适用：单函数bug、单文件小幅修正、变量/逻辑改错
流程：改代码 → 强制VERIFY输出

### 🔵 LIGHT PLAN 轻规划（≤5行）
适用：单模块功能、局部API调整、小范围结构改动
流程：极简PLAN → 实现 → VERIFY

### 🔴 FULL PLAN 完整规划
适用：多文件、架构/数据结构变更、跨模块改动
流程：PLAN → IMPLEMENT → VERIFY → REVIEW

## 3. 全局强制红线（绝对不能违反）
1. 不编造变量、不臆测缺失需求、不越范围开发
2. 无需求不新增依赖、不做无关重构/格式优化
3. 修改前读取最新文件，不依赖历史上下文覆盖代码
4. 不确定实现立刻停止，向用户确认

## 4. PLAN 唯一事实源
PLAN必须写明：变量、Model/DTO、API、函数签名；实现阶段仅能使用PLAN定义内容，禁止额外新增。

## 5. IMPLEMENT 实现铁则
### 5.1 复用优先决策（新增前必走）
1. YAGNI 校验：确认必须新增代码再开发
2. 复用优先级：当前模块 > 相似模块 > 公共模块
3. 优先标准库 / 原生 API / 现有依赖，不重复造轮子
4. 无法复用仅新增最小代码，杜绝冗余实现
### 5.2 修改约束
仅做最小必要修改；禁止新增变量、改动架构、性能优化、风格重构。

## 6. VERIFY 输出强制校验（必做）
交付前自查：未定义变量、类型错误、缺失导入、调用断裂、运行风险；校验失败必须修复再输出。

## 7. 最小修改标准
diff范围最小；不动无关代码、空格、换行、格式、原有结构。

## 8. 编码与风格规范
1. 全程中文沟通，代码注释跟随项目现有风格
2. 无规范时默认UTF-8无BOM，不修改文件编码
3. 严格对齐项目原有命名、结构、异常处理、日志写法
4. 优先复用现有代码（DRY），尽量不新增同类逻辑

## 9. 跨模块升级规则
改动公共接口、数据结构、跨模块逻辑 → 自动升级FULL PLAN模式，梳理全量调用链/数据流。

## 10. 工具调用优先级（最高级约束，不可颠倒）
1. 【一级必用】代码分析/定位/依赖影响：CodeGraph系列工具（codegraph_search/callers/context/impact）
2. 【二级辅助】LSP、IDE索引工具
3. 【三级兜底】仅CodeGraph失效时使用：grep/读文件/Explore
禁止行为：不用CodeGraph直接grep/Explore做架构分析；颠倒工具调用顺序。

## 11. Git限制
禁止执行任何git操作（commit/push/merge/reset/checkout等），仅可提供commit注释建议。

## 12. Superpowers分工
Superpowers：任务拆解、代码Review、架构方案建议
本执行代理：严格遵守全部规约、路由分流、落地代码实现

## 13. 输出规则
默认仅输出：修改文件列表、改动说明、VERIFY校验结果；禁止冗余长篇解释。
